# 貳、背景

## 一、研究或開發動機

**計畫背景**

日月潭為南投觀光核心，屬本府「觀光休憩產業」發展指標區域。年吸引逾 600 萬人次，水上活動（立槳 SUP、獨木舟、電動遊湖船）、環潭自行車、纜車搭乘與廟宇景點為主要體驗產品。雖然上述交通與活動普遍具有低碳屬性，但目前缺乏統一的量化憑證與跨業獎勵語言，業者之間各自銷售、遊客減碳行為無感、也無法與地方食品產業（生活工藝）形成情感高峰。

**面臨的問題**

1. **減碳行為無法量化與驗證**：遊客搭電動船、騎單車、住不備品民宿等低碳行為，目前僅能口頭宣稱，缺乏客觀證據，無法對外用於個人 ESG 敘事或企業永續報告。
2. **跨業合作缺乏統一集點語言**：住宿、單車、餐飲、烘焙業者各自發紅利點數，無互通性，無法形成日月潭整體品牌。
3. **景點打卡缺乏可靠驗證方式**：純粹拍照上傳容易造假，但商業級 AI 影像辨識 API 成本高（Google Vision 約 USD 1.5／1000 次）、誤判時缺乏人工救濟途徑，不適合中小型業者長期運營。
4. **缺少完成旅程之情感儀式**：現有遊程結束即結束，缺少可記憶、可分享、可回訪的旅程終點儀式。

**市場與使用者需求**

- 遊客端：年輕家庭與重視永續價值之旅人，願意為「證據化」的低碳旅程付費。
- 業者端：中小型觀光業者需要低成本之數位轉型方案，不願自建 App。
- 政府端：南投縣政府推動觀光永續轉型，需具體可量化之示範案例。

**未來對客戶、使用者產生之效益**

- 遊客：取得可驗證之個人減碳成就與紀念麵包，可分享至社群。
- 業者：共用集點系統、提升客單價與回訪率、納入本縣永續旅遊示範名單。
- 政府：取得南投首個可量化之低碳旅遊數據，作為 SDG 11／12／13 政策成果。

## 二、創新性說明

本計畫之創新核心在於**把影像驗證分為三層降級處理，避開單一 AI 模型的高成本與黑箱問題**，具體創新點如下：

1. **三段式照片驗證鏈（EXIF GPS → 感知雜湊 → 人工覆核）**
   - 第一層：讀取照片 EXIF 中的 GPS 座標，以 Haversine 公式計算與景點座標之距離，未超過設定半徑（150~200 公尺）即通過。
   - 第二層：當照片不含 GPS 資訊（iOS 分享常剝除 EXIF），自動載入景點參考照，雙方各縮至 8×8 灰階，以平均雜湊（aHash）計算 64 位雜湊碼，漢明距離 ≤ 14 即通過。
   - 第三層：兩層皆不通過則送入人工覆核佇列，由營運方判讀後核准發點或駁回，避免誤殺誠實遊客。
   - **本機制既趨近於零 API 成本，又保留防偽能力與人工救濟，適合中小型業者長期運營。**

2. **LINE LIFF 零下載整合**：使用者無需下載專屬 App，於 LINE 原生介面即可完成掃 QR、拍照驗證、兌換核銷、社群分享全流程。對應本府鼓勵之 **AI 智慧應用** 與友善使用體驗。

3. **跨業集點聯盟之統一獎勵語言**：將住宿、體驗、餐飲、烘焙四類業者之行為統一換算為減碳點數（整合環境部排放係數），首創日月潭地區跨業統一集點機制。

4. **稀缺性兌換機制**：減碳麵包為非賣品（not for sale），僅致贈完成旅程者。以行為經濟學之稀缺性理論強化分享動機與品牌記憶。

| 項目 | 指標或規格 | 功能與應用 |
|------|----------|----------|
| 三段式照片驗證 | GPS 距離誤差 ≤ 200 m、aHash 漢明距離 ≤ 14/64、人工覆核平均處理時間 ≤ 24 小時 | 景點打卡照片之真偽驗證，免付費商業 AI API |
| LINE LIFF 平台 | LIFF SDK 2.22+；響應時間 ≤ 2 秒；支援 iOS 14+／Android 8+ | 零下載進入點，整合掃碼、拍照、兌換 |
| 動態簽章 QR | 每業者最多 3 組 QR（HMAC-SHA256＋timestamp，時效 10 分鐘） | 防偽打卡、防跨店濫用 |
| 碳足跡演算 | 採環境部 114 年排放係數 v2.0 | 每一筆打卡產出 CO₂e 減碳量 |
| 營運儀表板 | 日活躍、點數累積、兌換率、業者貢獻等 10+ 指標、資料匯出 CSV | 計畫 KPI 追蹤、期中期末報告 |
| 人工覆核後台 | 單頁列表＋照片預覽＋EXIF／pHash 診斷資料 | 覆核員 < 30 秒處理一筆 |
| 安全性 | 照片加密傳輸（TLS 1.3）；個資保存符合個資法 | 符合 LINE 平台規範 |
| 合作業者家數 | 15 家（住宿 4、體驗 5、餐飲 4、景點 5） | 跨業聯盟結案指標 |

## 二、技術／產品創新比較

| 項目 | 國內外既有水準 | 本計畫競爭優勢 |
|------|--------------|-------------|
| 景點打卡驗證 | 多採「簽到即算」（日月潭電子戳章）、或商業 AI 影像辨識（Google Vision、Azure Computer Vision） | 三段式驗證兼顧防偽與成本；每張成本趨近於 0；失敗有人工救濟 |
| 跨業集點 | 多以紅利點數綁定單一品牌 | 四業別統一碳點數；非同質性行為轉換為共通語言 |
| 終點兌換 | 多為折扣券、紀念品 | 稀缺性「不可販售」碳麵包，建立行為經濟誘因 |
| 平台介面 | 多需下載專屬 App | LINE LIFF 零下載，對中高齡遊客友善 |
| 營運成本 | 自建 App 年維護 50 萬元以上 | 雲端服務月費 3 千元內，中小型業者可負擔 |

4. **技術商品化時程縮短**：開源核心演算法之免費版本，降低縣內其他業者進入門檻。
5. **引導投資**：帶動合作業者合計投資設備／整修經費約 200 萬元（掛牌、導入裝備等）。
